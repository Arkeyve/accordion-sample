Business Leading Practices

  1. Creating invoices through the Supplier Portal​.
  2. Routing non-PO invoices for approval through job level hierarchy based on DoA​.
  3. Enforcing no-PO, no-Pay policy​.
  4. Use of OCR or 3rd party imaging solutions which can automate the process of paper based invoice entry.
  5. Maximize use of recurring invoices templates and distribution sets for non-purchase order (PO) invoices
  6. Invoice entry, invoice approval and payment processing roles should be clearly segregated
  7. Laverage Shared service model.
  8. Invoices from external sources should be brought in using integration methods like EDI, XML, FBDI, etc.
  9. Use purchasing solutions that generate ERS (Electronic Receipts Settlements)

Application Leading Practices

Below is the suggested practice:
  1. Limit manual invoice creation by working with suppliers on interface or OCR options.
  2. Use appropriate description for invoice header/line description. Description will be visible during invoice approval notification.
  3. For non-merchandise expenses, freight and miscellaneous expenses are usually allocated across invoice lines.
  4. Use recurring invoice feature for invoices that have static amount across multiple periods.
  5. Use distributions sets for accounting distribution across multiple locations or departments.
  6. Use show available prepayment options for system to display available prepayments while entering standard invoice.
  7. Use Permanent prepayment feature to deny application of invoices to prepayment.
  8. For imported invoices use automated reconciliation process in interfaces to match batch totals.
  9. Invoices related to goods should be matched 3-way. Invoices related to services should be matched 2-way.
  10. Deferred expenses should be used when multi-period prepayment exists but supplier does not send a monthly invoice.
  11. Invoice entry, invoice approval and payment processing roles should be clearly segregated.
  12. Use imaging solution which can automate the process of invoice entry.
